## Remote Access Requirements

### Balena Connectivity

Fizyr utilizes Balena for deployment, monitoring, and remote maintenance. This cloud solution enables:

- Software updates and package deployment
- Remote system monitoring
- Configuration management
- Log access and analysis

#### Network Requirements

The following ports must be accessible:

| Port | Protocol | Purpose                  |
| ---- | -------- | ------------------------ |
| 53   | UDP      | DNS resolution           |
| 123  | UDP      | NTP time synchronization |
| 443  | TCP      | HTTPS connections        |

**Required Domains:**

- \*.balena-cloud.com

#### Bandwidth Requirements

- Update Size: 20-40 GB
- Monthly Data Transfer: ~100-200 GB for logs and analysis
- Backup Data: 1-2 GB per maintenance session

### Remote Access Scenarios

1. **Configuration**

   - Duration: 1-2 hours
   - Purpose: System setup and testing after updates/installation

2. **Maintenance**

   - Purpose: Regular data backup
   - Data Volume: 1-2 GB

3. **Inspection**

   - Frequency: Monthly
   - Data Volume: 100-200 GB
   - Purpose: Log retrieval and analysis

4. **Support & Monitoring**
   - Duration: Variable based on issue
   - Some contracts require continuous connectivity

## Support Structure

### Standard Support

Fizyr provides third-line support by default, which includes:

- Remote troubleshooting
- System optimization
- Software updates
- Performance monitoring

### Offline Support Options

For environments with limited connectivity:

- System updates via USB
- Algorithm tuning
- Neural network updates
- Data backup management

**Note:** Offline support requires physical access to the Fizyr PC with keyboard, mouse, and screen.
